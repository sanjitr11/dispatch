# agent-env — Startup Context

## What We're Building
Persistent specialized agent environment for solo founders — built on Claude Code with automatic context management, task routing, and reliability primitives from day one.

## Stage
Building the first version — Ship to learn. Speed > polish. Every decision is reversible.

## Tech Stack
TypeScript, Node.js 24, tsx, node:sqlite, @clack/prompts, Context-RAII (SQLite session lifecycle)

## Ideal Customer
Solo founders and early-stage startup teams using Claude Code who waste hours manually writing CLAUDE.md files, configuring subagents, and managing context across sessions.

## Current Priorities
1. Ship Sprint 1 MVP end-to-end
2. run init on 5 pilot projects from the AI founder network
3. collect feedback that changes the product

## Biggest Bottleneck
Agents start blank every session — no persistent memory of decisions, codebase conventions, or what's been investigated.

## Agent Team

This project uses a specialized agent environment. Use these agents when
delegating work via the Task tool, or invoke them directly with a slash command.

| Command | Agent | Use for |
|---------|-------|---------|
| `/coding` | Coding Agent | Implementation, bugs, refactoring, code review, deploys |
| `/research` | Research Agent | Decisions, competitor analysis, library evaluation, tradeoffs |
| `/marketing` | Marketing Agent | Copy, content, messaging, positioning, pitch, launches |
| `/ops` | Ops Agent | Planning, tooling, admin, integrations, everything else |
| `/route` | Auto-router | Describe a task — the environment picks the right agent |

## Routing Rules
- Code / architecture / debugging → `/coding`
- Decisions / investigations / comparisons → `/research`
- Copy / content / messaging / positioning → `/marketing`
- Ambiguous → `/route` and let the environment decide
